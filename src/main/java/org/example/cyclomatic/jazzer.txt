Для тестирования из предложенных вариантов мной был выбран Jazzer и его зависимость для интеграции с JUnit. В качестве
примера взял небольшое приложение Ai-агента, которое получает сообщение пользователя и применяет к нему различные методы
промпт-инженерии (например Re-reading). Приложение всего на 100-150 строк кода. Фазз-тестированием было выявлено OutOfMemory.
Ошибка произошла из-за отсутствия ограничения на размер поступаемого пользовательского сообщения и многократного его дублирования
для составления итогового промпта от пользователя. Трудности возникли прежде всего при выборе самого фаззера, даже с учетом,
что для java выбор был предопределен, хотелось побольше разобраться и из-за этого долго не мог определиться и бегал
по различным документациям в поисках идеального для первой пробы варианта. После выполнения задания кажется, что
для серьезного применения фаззинга недостаточно поверхностного ознакомления, поэтому после сдачи задания планирую уделить
этой теме еще время, чтобы окончательно разобраться. Кажется с учетом нынешних ai-реалий такой вид тестирования будет
только расти в ценности при написании плохого кода.
